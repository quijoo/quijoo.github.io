游戏中的Attribute系统
2025-12-13 06:32:43

游戏属性系统

设计意图

游戏中有很多角色属性, 他们之间存在依赖关系，组成一个复杂的树形结构（非网状），并且需要在编辑器中编辑结构和叶节点的初始值。

并且，当角色实例化后应该拷贝该角色的属性树，并且所有修改都只在实例中生效。所以以节点树的形式保存属性树，它们将会随着 PackedScene 的实例化而复制。

但是，有的属性是全局的，他需要被实例化的节点树引用和修改，所以应该用 Resource 表示全局的属性。由此节点树和资源树构成了编辑器时和运行时的两个并行树结构，而它们之间通过 ReferenceNode 单向引用。

系统结构

节点:
1. ValueNode 值节点基类
2. IntNode： int 值节点
3. FloatNode： float 值节点
4. ReferenceNode： 引用节点（用于引用 VauleResource）
5. FormulaNode： 公式节点（用表表示多个值组成的表达式）

资源：
1. ValueResource 值资源
2. IntResource int 值资源（编辑器内创建和编辑）
3. FloatResource float 值资源（编辑器内创建和编辑）
4. FormulaResource 公式资源（用于表示多个值资源组成的表达式）

值资源在编辑器中组成一颗属性树, 这个树用于表示与实例无关的游戏属性值，例如：

  1. 角色血量，角色经历，角色金钱，角色库存 (单例相关)
  2. 当前关卡难度，当前世界风力，当前重力 （世界属性）
  3. …
    使用资源表示这些变量能 定义默认值、各个实例方便引用、各个实例修改能立即生效并通知

值的变化

当某个叶（资源/值）的值发生变化时, 会发出变化信号, 该信号会沿着树向各个根节点传播，这使得树上任何一个值的修改都会被通知，包括公式节点（公式变量发生变化时重新计算公式，公式结果发生变化后发出信号）

应用

  1. 在那些需要在 HUD 中显示的值可以用值资源显示，分别由实例节点和UI节点引用
  2. 全局进度存档，有许多实例引用同一个进度资源
  3. 数值难度调整全局生效，并且经由多个公式节点传播到各个实例
  4. 可以很方便的实现属性存档功能

优化

使用 Node 节点是完全没必要的, 因为资源 Resource 可以 LocalToScene, 并且同一个 PackedScene 中的不同资源在实例化LocalToScene 时会保留他们之间的引用（这就是期望节点完成的功能！！！），太完美了妈妈咪呀。
